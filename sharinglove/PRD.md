# 4. 페이지별 PRD

## 4.1 사랑나눔복지센터 > 소개

### 페이지 목적

* 센터의 정체성, 가치, 운영 배경을 전달하여 신뢰 형성

### 콘텐츠 구성(섹션)

1. **센터 로고/상징 설명**

* 원고: “하트 안에 사람과 이를 감싸는 손…” 문단 그대로 노출

2. **슬로건/핵심 메시지**

* 타이틀 예: `사랑과 책임으로 함께 걸어온 길`
* 원고: “사랑나눔복지센터는 2011년 10월 5일…” 문단

3. **사업 운영 범위/특징**

* 원고: “장애인활동지원사업을 중심으로… 방문목욕사업…” 문단
* 강조 문구(한 줄): “우리는 제도와 숫자보다 사람의 삶을 먼저 생각합니다.”

### 기능 요구사항

* 관리자에서 소개글(텍스트/이미지) 수정 가능(에디터)
* 주요 문단별 제목/본문 분리 저장(추후 개편 대비)

### 수용 기준(Acceptance Criteria)

* 소개 페이지에 상단 타이틀/본문이 모바일에서도 가독성 있게 줄바꿈 처리됨
* 관리자가 텍스트를 수정하면 즉시 반영됨(캐시 정책은 개발 표준에 따름)

## 4.2 사랑나눔복지센터 > 오시는길

### 페이지 목적

* 방문/내방에 필요한 “지도 + 대중교통 + 자가용 경로 + 주소” 제공

### 콘텐츠 구성(섹션)

1. **지도**

* 네이버 지도 링크 버튼: `https://naver.me/xl0Ne8n0`
* (권장) 지도 임베드가 가능하면 임베드, 어려우면 “지도 보기” 버튼으로 처리

2. **주소**

* `충남 천안시 서북구 성거읍 천흥4길 72-7`

3. **대중교통(시내버스) 안내**

* 종합터미널: 200, 201, 121 (천흥리 하차)
* 천안역: 200, 201 (천흥리 하차)
* 천안아산역: 15 (종합터미널) → 200, 201, 121 (천흥리 하차)
* 성환역: 160, 164 (입장양진유통 하차) / 200, 201 (삼일아파트 하차)

4. **자가용(출발: 천안시청) 경로 안내**

* 원고 제공 루트 그대로 노출
* “📌 참고” 문구 포함
* “골목길… 옆에 바로 보입니다.” 안내 포함

### 기능 요구사항

* 지도 링크/주소/교통 안내를 관리자에서 수정 가능
* “복사하기(주소)” 버튼(선택 기능, 있으면 편리)

### 수용 기준

* 모바일에서 버스 노선이 줄바꿈되어 읽기 쉬움
* 지도 링크 버튼이 항상 상단(또는 주소 근처)에 노출되어 클릭 1회로 이동 가능

## 4.3 사업소개 > 장애인활동지원

### 페이지 목적

* 제공 서비스(활동보조/방문목욕)와 센터 운영 철학을 명확히 안내

### 콘텐츠 구성(섹션)

1. **서비스 개요**

* 서비스 항목을 **2가지 탭 또는 2개 카드**로 고정 표기

  * `활동보조`
  * `방문목욕`

2. **활동보조(상세)**

* 외부 참고 링크 표기: `https://www.cdaws.or.kr/59`
* (운영 정책) 링크는 “자세히 보기” 버튼으로 제공
* 내부 페이지에는 최소한의 요약 문구를 넣고, 상세는 참조 링크로 유도(원고/운영 편의)

3. **방문목욕(정의/조건)**

* 원고 그대로 구성:

  * “활동지원인력인 요양보호사가…” 정의
  * 이동목욕(차량 내) / 가정 내 목욕(욕조·펌프·호스릴 등) 구분
  * 예외 조건(차량 온수 사용 불가 + 수급자 동의 시) 문구 포함

4. **센터 운영 철학/성과 메시지(신뢰 섹션)**

* 원고: “장애인의 일상이 보다 존엄하고…” ~ “동반자가 되겠습니다.” 문단 전체 노출

### 기능 요구사항

* 관리자에서 “활동보조 요약문/링크”, “방문목욕 상세문” 수정 가능
* 서비스 영역은 추후 추가를 고려해 구조 확장 가능(현재는 2개 고정)

### 수용 기준

* 화면에서 “활동보조/방문목욕” 두 서비스가 한눈에 구분됨
* 외부 링크는 새 창 열기(권장) 또는 동일 창(정책 선택)로 일관되게 처리

### 수용 기준

* 대표명과 본문이 명확히 분리되어 보임(타이포 계층)

* 2015.01.01 …

### 기능 요구사항

* 관리자가 연혁 항목을 추가/수정/삭제/정렬(최신순/과거순) 가능

### 수용 기준

# 4) 공지·소식 게시판 PRD (프론트 + 관리자)

## 4.1 공통 요구사항(4개 게시판 공통)

### 4.1.1 프론트(사용자) 기능

* **목록 페이지**

  * 표시: 제목, 등록일, (선택) 첨부 아이콘, (선택) 조회수
  * 페이징: 기본 10개/페이지(설정값으로 변경 가능)
  * 검색: 제목/내용 키워드 검색
  * 정렬: 최신순 기본

* **상세 페이지**

  * 제목, 등록일, 본문, 첨부파일 다운로드
  * 이전글/다음글 이동
  * URL 공유 가능(고정 링크)

### 4.1.2 어드민(관리자) 공통 기능

* 게시글 **등록/수정/삭제**
* 공개 상태: `공개/비공개`
* 첨부파일 업로드/삭제
* 목록에서 검색/필터/정렬
* 마지막 수정일 자동 기록

### 4.1.3 공통 데이터 모델(권장)

* Post

  * id (PK)
  * boardType: `NOTICE | BUDGET | RESOURCE | GALLERY`
  * title (필수, 1~100자)
  * content (WYSIWYG, 정책에 따라 필수/선택)
  * isPublished (공개 여부)
  * createdAt, updatedAt
  * views (선택)

* Attachment

  * id (PK)
  * postId (FK)
  * filenameOriginal
  * fileUrl
  * fileSize (선택)
  * createdAt

> 갤러리는 이미지도 Attachment로 통합 가능(이미지 타입 플래그) 또는 별도 GalleryImage 테이블로 분리(선택)

### 4.1.4 파일 정책(권장)

* 이미지: jpg/png/webp (10MB/파일)
* 문서: pdf/doc/docx/xls/xlsx/hwp/zip 등 (20MB/파일)
* 업로드 개수 제한: 기본 10개/게시물(설정 가능)

## 4.2.1 공지사항(Notice)

### 목적

* 이용자/활동지원사 대상 주요 공지 전달

### 프론트

* 목록: 고정 공지(핀) 상단 노출 후 일반 공지 최신순
* 상세: 첨부 다운로드 가능

### 어드민 입력 필드

* 제목(필수)
* 본문(필수, WYSIWYG)
* 첨부(선택, 다중)
* 상단 고정(isPinned) 토글
* 공개/비공개

### 검증/정책

* 제목 1~100자
* 본문 최소 1자
* 고정 공지 최대 N개(권장 3개) 옵션

---

## 4.2.2 예산 및 결산서(Budget & Settlement)

### 목적

* 예산서/결산서 등 대외 공개 자료를 연도별로 제공

### 프론트

* 목록: 기본 정렬 `연도 최신순` → 동일 연도 내 `등록일 최신순`
* 필터(권장): 연도 선택, 구분(예산/결산)
* 상세: 첨부 중심(다운로드 버튼 강조)

### 어드민 입력 필드

* 연도(year) (필수, 숫자)
* 구분(type) (필수): `예산` / `결산`
* 제목(필수) *(자동 생성 옵션 가능: “2024년 예산서” 같은 형태)*
* 본문(선택) *(설명 필요하면 작성)*
* 첨부파일(필수, 최소 1개)
* 공개/비공개

### 검증/정책

* year: 2000~2100 범위(권장)
* 첨부 1개 이상 필수(PDF 권장)
* 동일 연도/구분 중복 업로드 허용(가능)

---

## 4.2.3 자료실(Resources)

### 목적

* 서식, 안내문, 신청서, 참고자료 업로드/다운로드 제공

### 프론트

* 목록: 최신순 + 검색
* 상세: 첨부 다중 다운로드

### 어드민 입력 필드

* 제목(필수)
* 본문(선택)
* 첨부파일(선택, 다중)
* 공개/비공개

### 검증/정책

* 제목 필수(1~100자)
* 첨부 없이도 등록 가능(공지형 자료실 글 허용)

---

## 4.2.4 갤러리(Gallery)

### 목적

* 활동 사진, 행사 기록 등 이미지 중심 콘텐츠 제공

### 프론트

* 목록: 썸네일 그리드(카드형), 페이징
* 상세: 이미지 슬라이드 또는 세로 나열(선택)
* 대표 썸네일: 첫 이미지 자동 또는 대표 선택

### 어드민 입력 필드

* 제목(필수)
* 본문(선택)
* 이미지 업로드(필수 권장, 다중)
* 이미지 정렬(드래그 정렬 권장)
* 대표 이미지 지정(선택)
* 공개/비공개

### 검증/정책

* 이미지 최소 1개(권장: 필수로 구현)
* 이미지 파일만 업로드 허용
* 업로드 이미지가 없으면 저장 불가(필수로 할 경우)

# 5) 어드민 화면 구성(최소 요구)

## 5.1 어드민 메뉴

* `/admin`

  * 공지사항 관리
  * 예산 및 결산서 관리
  * 자료실 관리
  * 갤러리 관리

## 5.2 공통 화면(각 게시판 동일 패턴)

### (1) 목록 화면

* 테이블 컬럼(기본)

  * 제목(또는 예산/결산은 “연도/구분/제목”)
  * 공개상태
  * 등록일/수정일
  * (선택) 조회수
  * 수정 / 삭제 버튼
* 검색: 키워드(제목/내용)
* 필터:

  * 공개상태(전체/공개/비공개)
  * 예산/결산은 추가로 연도/구분 필터
* 버튼: “새 글 작성”

### (2) 작성/수정 화면

* 입력 폼(게시판별 정의 적용)
* 버튼: 저장 / 임시저장(선택) / 취소
* 파일 업로드 영역: 첨부/이미지 리스트 + 개별 삭제

### (3) 삭제 정책

* 1차: 삭제 확인 팝업
* 권장: 소프트 삭제(복구 가능) 또는 하드 삭제(단순) 중 택1
  *(단순 운영이면 하드 삭제도 가능)*

---

# 6) 수용 기준(Acceptance Criteria)

* 관리자는 4개 게시판 모두에서 **등록/수정/삭제/공개전환**이 가능해야 함
* 사용자는 공개 글만 열람 가능(비공개는 404 또는 목록/상세 미노출)
* 예산/결산서는 **첨부 최소 1개** 조건을 만족하지 않으면 저장 불가
* 갤러리는 이미지 중심으로 목록 썸네일이 정상 노출되어야 함
* 첨부파일은 상세 화면에서 다운로드가 정상 동작해야 함
